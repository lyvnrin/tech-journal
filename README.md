# lavanya.log

A record of things I built, broke, fixed, and figured out. Not a portfolio or a blog. Something closer to a logbook - the kind kept not because anyone is watching, but because it is useful to have written things down.

Each entry covers something I actually worked on: a prototype, a script, a tool, an experiment. The format is loose by design. There is a diary note at the top, then the technical detail, then a reflection. Some entries will be polished, though most will not

## Structure

```
/YYYY-MM
  DD-slug.md
```

Entries are named by date so they sort chronologically without an index. The most recent is always at the bottom.

## What counts as an entry

The bar is: *did I learn something, or hit a wall?*

That includes CLI scripts, Lovable prototypes, OpenCV experiments, half-finished ideas, things that worked immediately, and things that took four hours and one embarrassingly obvious fix. It does not need to be impressive, honestly

## Why this exists

Because scattered repos with no narrative between them are hard to read - including by the person who wrote them. This is an attempt to thread them together without flattening them into a highlights reel.
